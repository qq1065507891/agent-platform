# 已完成功能与模块清单（基于代码与开发文档对照）

> 本清单依据 `开发文档.md` 与当前代码实现整理，仅包含**已在代码中落地**的功能与模块。

## 1. 前端核心界面（MVP）

### 1.1 前端工程与基础框架
- 已初始化 `Vue 3 + TypeScript + Vite` 前端工程（frontend）。
- 完成路由与状态管理初始化（vue-router、pinia）。
- 接入 Arco Design 组件库与图标库。
- 配置 Vite 代理，将 `/api` 请求转发到 `http://127.0.0.1:8000`。
- 封装 Axios 网络层，支持 Token 注入、统一响应解析与 401/403 自动跳转登录。

### 1.2 全局布局与登录
- 完成后台标准布局（顶部导航 + 侧边栏 + 内容区）。
- 登录页面支持账号密码登录，调用 `POST /api/v1/auth/login`，成功后保存 Token 并进入主页。
- 未登录访问受保护页面自动跳转登录。

### 1.3 智能体市场
- 智能体市场列表页支持搜索与栅格化卡片展示。
- 点击智能体卡片调用 `POST /api/v1/conversations` 创建会话并跳转对话页。

### 1.4 对话界面
- 对话页完成基础布局：左侧历史列表占位 + 右侧聊天区。
- 支持消息气泡展示、输入框与发送按钮。
- 调用 `POST /api/v1/conversations/{id}/messages` 发送消息并处理 Loading 状态。

## 2. 后端基础能力

### 1.1 服务框架与统一响应
- FastAPI 应用已搭建，API 基础路由前缀为 `/api/v1`。
- 统一成功响应结构（`code/message/data`）。
- 全局异常处理（参数校验、HTTP 异常、未捕获异常）。

### 1.2 数据库与配置
- SQLAlchemy 基础连接与 Session 管理。
- `.env` 加载，支持通过 `DB_URL` 指定数据库连接。
- 基础配置模块（LLM 网关、模型参数）。

### 1.3 认证与安全
- JWT 访问令牌生成与校验。
- 基础密码哈希与校验（SHA256）。
- Bearer Token 认证依赖（未认证拒绝访问）。

## 2. 用户与权限（MVP 范围已部分实现）

### 2.1 用户登录与登出
- `POST /auth/login`：账号登录，返回 access_token 与用户信息。
- `POST /auth/logout`：登出接口（当前为成功返回）。

### 2.2 用户管理（管理员权限）
- `GET /users`：分页查询用户列表（支持 keyword 过滤）。
- `POST /users`：创建用户。
- `PUT /users/{id}`：更新用户。
- `POST /users/import`：批量导入用户。
- 用户角色与状态字段支持 `admin/user/manager` 与 `active/disabled`。

> 说明：当前为 RBAC 的最小形态（仅管理员可操作用户管理）。

## 3. 智能体管理（MVP）

### 3.1 智能体 CRUD
- `GET /agents`：分页查询智能体列表（支持 keyword 与 is_public 过滤）。
- `POST /agents`：创建智能体。
- `GET /agents/{id}`：获取智能体详情。
- `PUT /agents/{id}`：更新智能体（仅创建者或管理员）。

### 3.2 智能体数据模型
- Agent 模型包含：名称、描述、Prompt 模板、skills、是否公开、状态、owner_id。
- skills 字段支持 JSON 列表并进行基础校验。

> 说明：发布/下架流程（`publish/unpublish`）尚未在代码中实现。

## 4. 对话与会话（MVP）

### 4.1 会话管理
- `POST /conversations`：创建会话。
- `GET /conversations/{id}`：获取会话详情。
- 会话仅允许所属用户访问。

### 4.2 消息发送与多轮对话
- `POST /conversations/{id}/messages`：发送用户消息并返回助手回复与 trace_id。
- 会话消息存储在数据库 JSON 字段中。
- 支持多轮上下文（历史消息拼接）。

## 5. Agent 编排与工具调用（MVP）

### 5.1 LangGraph 编排
- 使用 `StateGraph` 构建对话流程。
- 支持 AIMessage tool_calls 的工具分支与回路调用。

### 5.2 内置 Skills（已实现）
- `calculator`：四则运算工具。
- `current_time`：获取当前时间工具。

> 说明：开发文档中提到的 `web_search/text_summary/translator` 等工具尚未在代码中实现。

## 6. 数据模型与 Schema

### 6.1 已实现数据模型
- `User`：用户表。
- `Agent`：智能体表。
- `Conversation`：会话表。
- `Skill`：技能表（模型已定义，业务接口未落地）。
- `RequestLog`、`LLMUsage`、`SkillInvocation`、`EventLog`：已定义模型文件（但尚未接入业务流程）。

### 6.2 Pydantic Schema
- 用户、智能体、会话、技能、认证、通用响应等 Schema 已齐全。
